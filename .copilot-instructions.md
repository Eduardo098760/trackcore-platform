# 🎯 Padrões de Desenvolvimento - Trackcore Platform

## TypeScript & Type Safety

### ✅ SEMPRE fazer antes de qualquer commit:
1. **Executar verificação de tipos**: `npm run build` (TypeScript strict mode)
2. **Validar tipos em cada arquivo modificado**:
   - Verificar se todas as funções têm tipos explícitos de parâmetro e retorno
   - Garantir que nenhuma propriedade indefinida está sendo acessada
   - Usar `??` (nullish coalescing) quando apropriado
   - Evitar `any` - usar tipos específicos sempre

3. **No mínimo, antes de implementar mudanças**:
   - Ler os tipos relevantes em `/src/types/index.ts`
   - Validar se a estrutura esperada existe na interface
   - Adicionar imports necessários

### Validação de Tipos na Prática

```typescript
// ❌ ERRADO
const handleCreate = (data) => {
  createMutation.mutate(data);
};

// ✅ CORRETO
const handleCreate = (data: Omit<Device, 'id'>) => {
  createMutation.mutate(data);
};

// ❌ ERRADO
const position = device.attributes?.organizationId;

// ✅ CORRETO - verificar tipos disponíveis
const clientId = device.clientId;
const status = device.status ?? 'offline';
```

## Error Handling

```typescript
// ✅ Padrão correto
try {
  const result = await apiCall();
  return result;
} catch (error) {
  console.error('[FunctionName] Error:', error);
  toast.error('Mensagem amigável ao usuário');
  throw error; // Re-throw se necessário para tratamento acima
}

// Em React Query
const query = useQuery({
  queryKey: ['data'],
  queryFn: async () => {
    try {
      return await fetchData();
    } catch (error) {
      console.error('[fetchData] Error:', error);
      throw error;
    }
  },
  onError: (error) => {
    toast.error('Erro ao carregar dados');
  },
});
```
